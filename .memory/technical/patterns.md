# Patterns Library

Reusable code patterns with context, examples, and rationale.

Pattern names are noun phrases ("Safe First Helper", not "How to use safe_first").

---

## How to add a pattern

```markdown
## Pattern Name

**Context:** When to use this pattern

**Code example:**
```python
# minimal, concrete implementation
```

**Why it works:** Mechanism, not description
```

---

## Quality checklist for patterns

- [ ] Does it have a specific context (when to use)?
- [ ] Does it have working code?
- [ ] Does it explain WHY it works, not just what it does?
- [ ] Is the name a noun phrase?
- [ ] Is it non-obvious (not just "use a function instead of copying code")?
